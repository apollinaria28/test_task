# Задание 2: Проектирование API
### Интернет-магазин «Петрушка Зеленая»

---

## Описание задания

Интернет-магазин «Петрушка Зеленая» преуспевает, расширяется и в мобильном приложении решили создать новый экран, который будет отображать магазины партнёров.

**Что нужно сделать:**
1. Написать пример REST API запроса, который будет вызываться при переходе пользователя на данный экран.
2. Привести пример ответа этого REST API в соответствии с макетом. Формат — JSON. Учесть, что при клике на плашку магазина должен осуществляться переход по ссылке на внешний ресурс.

---

## Выполнение

## Что отображается на экране

Страница с карточками магазинов. Каждая карточка содержит:

| # | Поле | Описание |
|---|------|----------|
| 1 | Название | Наименование магазина |
| 2 | Тип доставки | `scheduled` — ближайшая, `express` — быстрая |
| 3 | Временной интервал | Рассчитывается на сервере на основе геопозиции |
| 4 | Логотип | URL изображения |
| 5 | Ссылка | Переход на внешний ресурс магазина |

---

## Логика расчёта временного интервала

Чтобы вычислить время доставки, сервер получает геопозицию пользователя через query-параметры — `latitude` и `longitude`. В базе хранятся координаты каждого магазина. На стороне сервера вычисляется предполагаемое время доставки и возвращается в готовой строке `delivery_time`.

---

## Авторизация

Страница доступна только авторизованным пользователям. Токен передаётся в заголовке каждого запроса:

```
Authorization: Bearer {token}
```

---

## Логика отображения типа доставки

Поле `delivery_type` — машиночитаемое значение. Фронтенд применяет стили на его основе:

| Значение | Отображение |
|----------|-------------|
| `scheduled` | «Ближайшая доставка» — чёрный текст |
| `express` | «Быстрая доставка» — синий текст |

> Фронтенд сам решает, какой цвет применить — сервер отвечает только за данные, не за стили.

---

## Пример запроса

```http
GET /petrushka/v1/?latitude=67.7558&longitude=47.6745&page=1&limit=10

Headers:
  Authorization: Bearer {token}
```

**Параметры запроса:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `latitude` | float | Широта геопозиции пользователя |
| `longitude` | float | Долгота геопозиции пользователя |
| `page` | int | Номер страницы (пагинация) |
| `limit` | int | Количество магазинов на странице |

## Описание полей ответа

| Поле | Тип | Описание |
|------|-----|----------|
| `page` | int | Текущая страница |
| `total` | int | Общее количество магазинов |
| `shops` | array | Массив карточек магазинов |
| `id` | string | Уникальный идентификатор магазина |
| `name` | string | Название магазина |
| `delivery_type` | string | Тип доставки: `scheduled` или `express` |
| `delivery_time` | string | Рассчитанный интервал доставки |
| `logo_url` | string | URL логотипа магазина |
| `store_url` | string | Ссылка для перехода на внешний ресурс |
